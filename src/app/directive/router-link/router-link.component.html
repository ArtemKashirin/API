RouterLink - атрибут: [routerLink] - применяется в качестве атрибута к любому блочному HTML-элементу, чаще к тегу а и делает этот элемент ссылкой, при нажатии на которую инициируется навигация по маршруту. Навигация открывает один или несколько маршрутизируемых компонентов в одном или нескольких местах на странице при помощи тега router-outlet. В [routerLink] передается путь отображения маршрута в URL-адресе.

Совместно с [routerLink] могут применяться дополнительные атрибуты:

  • queryParams?: Params | null - добавляет параметр запроса к сгенерированному URL-адресу(например: /user/bob?debug=true);
  • fragment?: string - добавляет фрагмент к сгенерированному URL-адресу(например: /user/bob#education);
  • queryParamsHandling?: QueryParamsHandling | null - возможность задать директиве обрабатывать параметры запроса по разному, при помощи допустимых значений: 'merge' - объединяет данные queryParams в текущие параметры запроса, preserve - сохраняет текущие параметры запроса;
  • preserveFragment: boolean - если задано значение true, фрагмент URL-адреса сохраняется для следующей навигации;
  • skipLocationChange: boolean - если задано значение true, выполняется навигация без сохранения нового состояния в истории;
  • replaceUrl: boolean - если установлено значение true, выполняется переход с заменой текущего состояния в истории;
  • relativeTo?: ActivatedRoute | null - задает корневой URI, используемый для относительной навигации;
  • state?: [k: string]: any; - определяемое разработчиком состояние, которое может быть передано в любую навигацию. После навигации завершается, маршрутизатор записывает объект , содержащий это значение вместе с navigationIdк history.state.
